## 修复内容总结

### 问题 1: 生成的 RF 用例名称包含空格 ✅ 已修复

**修复措施**:

1. **工作流文件** (`requirement-to-rf.md`):
   - 在 `skill_generation` 节点添加用例命名规范（强制）
   - 明确命名格式: `业务操作_具体场景`
   - 强制使用下划线 `_`，禁止使用空格
   - 提供命名转换规则和示例

2. **Skill 文件** (`test/SKILL.md`):
   - 新增"用例命名规范（强制）"章节
   - 定义命名转换规则: 空格替换为下划线
   - 提供转换示例表格
   - 列出正确和错误示例

3. **规范检查 Skill** (`rf-standards-check/SKILL.md`):
   - 新增用例命名规范检查项
   - 定义检查规则: 必须包含下划线，禁止空格
   - 提供检查方法和修正建议

**规范要求**:
```robotframework
# ✅ 正确
商户状态变更_正常变暂停
生成入网商户号_首次

# ❌ 错误
商户状态变更 正常变暂停  (包含空格)
```

### 问题 2: 未按标准目录结构生成文件 ✅ 已修复

**修复措施**:

1. **工作流文件** (`requirement-to-rf.md`):
   - 在 `skill_generation` 节点添加标准目录结构（强制）
   - 定义必须生成的 4 个文件:
     - `Settings.robot` - 套件设置和初始化
     - `Keywords.robot` - 用户关键字定义
     - `Variables.robot` - 变量定义
     - `<需求名称>_测试用例.robot` - 测试用例
   - 提供每个文件的模板内容
   - 定义文件引用关系

2. **Skill 文件** (`test/SKILL.md`):
   - 新增"目录结构规范（强制）"章节
   - 详细说明 4 个文件的用途和必需内容
   - 提供每个文件的模板代码
   - 定义生成要求和文件引用关系

3. **规范检查 Skill** (`rf-standards-check/SKILL.md`):
   - 新增目录结构规范检查项
   - 检查必须包含 4 个标准文件
   - 检查文件间的引用关系
   - 标记不符合规范的地方

**标准目录结构**:
```
<需求名称>_测试套件/
├── Settings.robot          # 套件设置和初始化
├── Keywords.robot          # 用户关键字定义
├── Variables.robot         # 变量定义
└── <需求名称>_测试用例.robot   # 测试用例
```

**文件引用关系**:
- Settings.robot → Keywords.robot, Variables.robot
- Keywords.robot → Variables.robot
- 测试用例文件 → Settings.robot

### 问题 3: 未处理参考用例的复用 ✅ 已修复

**修复措施**:

1. **工作流文件** (`requirement-to-rf.md`):
   - 新增 `skill_reference_analysis` 节点
   - 在流程图中添加参考用例分析步骤
   - 更新执行流程，将参考用例分析作为第 4 步
   - 添加输入参数 `reference_cases_dir`
   - 定义分析内容、复用原则和输出

2. **Skill 文件** (`test/SKILL.md`):
   - 新增"参考用例复用规范（强制）"章节
   - 定义复用原则: 优先复用、引用复用、谨慎修改、禁止重复
   - 提供分析步骤和匹配复用建议表
   - 提供复用示例和生成要求

**复用原则**:
| 原则 | 说明 |
|------|------|
| 优先复用 | 功能完全匹配的关键字直接复用 |
| 引用复用 | 通过 Resource 引用已有文件 |
| 谨慎修改 | 不要轻易修改 Variables.robot 中的变量 |
| 禁止重复 | 禁止重复实现已有功能的关键字 |

**匹配复用建议**:
| 匹配程度 | 处理方式 | 示例 |
|----------|----------|------|
| 完全匹配 | 直接复用，通过 Resource 引用 | `商户详情查询` 已存在，直接引用 |
| 部分匹配 | 修改后复用，说明差异点 | `V4商户状态变更` 可复用 `商户状态变更` 逻辑 |
| 不匹配 | 新建关键字 | 全新业务功能 |

**更新后的执行流程**:
1. 需求获取 - 从 TAPD 拉取需求内容
2. 场景识别 - 识别测试场景
3. 测试点识别 - 识别具体测试点
4. **参考用例分析** - 分析参考用例，提取可复用的关键字和变量 ⭐新增
5. 接口文档 - 从 YAPI 获取接口文档
6. 用例生成 - 生成 RF 测试用例（复用已有内容）
7. 质量保证 - RF 质量保证 Agent 检查用例质量
8. 规范检查 - 检查用例规范
