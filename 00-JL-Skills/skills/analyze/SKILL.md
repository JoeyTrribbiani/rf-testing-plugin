# 代码深度解析技能

## 技能概述

本技能提供测试工程师视角的代码分析能力，通过结构分析、流程分析和影响面分析三个维度，帮助测试工程师理解代码变更，识别测试点和风险。

## 适用场景

- 从 GitLab/GitHub 获取代码变更后进行分析
- 识别改动点和测试范围
- 评估代码变更对现有测试的影响
- 生成测试场景和测试点建议

## 分析维度

### 1. 结构分析（3步骤）

#### 步骤1: 技术栈识别
- 识别项目使用的编程语言、框架、依赖库
- 识别测试相关技术（测试框架、Mock工具等）

#### 步骤2: 实体ER图分析
- 识别核心业务实体和数据模型
- 分析实体之间的关系

#### 步骤3: 接口入口识别
- 识别对外暴露的API接口
- 识别内部服务间调用接口

### 2. 流程分析（3步骤）

#### 步骤1: 调用链分析
- 分析代码的调用链路
- 识别关键路径和分支路径

#### 步骤2: 时序分析
- 分析代码执行时序
- 识别异步操作和并发场景

#### 步骤3: 复杂逻辑识别
- 识别条件分支、循环、递归等复杂逻辑
- 识别边界条件和异常情况

### 3. 影响面分析（3步骤）

#### 步骤1: 依赖引用分析
- 分析代码的依赖关系
- 识别被改动代码的引用方

#### 步骤2: 数据影响分析
- 分析对数据模型的影响
- 识别数据迁移需求

#### 步骤3: 风险评估
- 评估改动的影响范围
- 识别高风险区域和回归测试重点

## 使用方式

### 完整分析
```
skill "analyze" "对 [代码路径] 进行完整代码分析"
```

### 单维度分析
```
skill "analyze" "对 [代码路径] 进行结构分析"
skill "analyze" "对 [代码路径] 进行流程分析"
skill "analyze" "对 [代码路径] 进行影响面分析"
```

## 输出规范

分析报告应包含：
1. **执行摘要** - 关键发现和风险等级
2. **详细分析** - 按三个维度展开
3. **测试建议** - 推荐的测试场景和测试点
4. **风险清单** - 需要重点关注的回归测试区域
