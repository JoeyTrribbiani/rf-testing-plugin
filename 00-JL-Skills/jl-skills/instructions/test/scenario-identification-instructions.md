# 场景识别指令

指导 AI 如何从需求文档中识别测试场景和测试点。

## 目标

从 TAPD 需求内容中识别：
1. **测试场景**：可测试的业务流程
2. **测试点**：每个场景需要验证的功能点
3. **边界条件**：正常、异常、边界场景

## 处理流程

1. **分析需求文本**
   - 提取核心功能描述
   - 识别前置条件
   - 识别操作步骤
   - 识别预期结果

2. **分类测试点**
   - 按功能模块分组
   - 按测试类型分组（功能测试、接口测试、数据测试）
   - 识别自动化可行性

3. **设计测试场景**
   - 正常流程场景
   - 异常处理场景
   - 边界值测试场景
   - 性能测试场景（如需要）

## 输出格式

```markdown
## 测试场景 1: [场景名称]

**需求描述**: [来自需求文档的描述]

**测试点**:
1. [测试点1]
2. [测试点2]
3. [测试点3]

**自动化可行性**: 是/否

---

## 测试场景 2: [场景名称]
...
```

## 关键词提示

关注以下关键词来识别测试场景：
- "应该"、"必须"、"需要"
- "当"、"如果"、"否则"
- "确保"、"验证"
- "支持"、"不支持"
- "正常"、"异常"、"错误"
- "最大"、"最小"、"空"
- "超时"、"并发"、"批量"

## 注意事项

1. 不要遗漏边界条件
2. 区分用户需求和系统需求
3. 识别性能相关需求
4. 标记需要特殊环境的测试场景
