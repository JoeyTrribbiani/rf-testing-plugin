# 商户系统业务规范

## 文档说明

本文档描述商户系统相关的业务编写规范和模式，包括存量商户和 V4.0 商户系统。商户系统是业务模块之一，其他业务模块（如订单系统、用户系统等）应有独立的业务规范文档。

**全局规范索引：** [Robot Framework 编写规范.md](../Robot Framework 编写规范.md)

---

## 1. 商户系统概述

商户系统包含以下核心业务模块：

| 模块 | 说明 |
|------|------|
| 存量商户 | 旧版商户系统，直接调用变更接口 |
| V4.0 商户 | 新版商户系统，基于申请单和审核流程 |

## 2. 存量商户业务模式

### 2.1 变更业务

**模式：** 修改-验证-恢复

```robotframework
*** Test Cases ***
商户状态变更-正常变暂停
    [Documentation]    【预置条件】已存在状态为"正常"的商户 【操作步骤】直接调用变更接口修改状态，查询验证，恢复原始状态 【预期结果】商户状态成功变更，验证通过后恢复原状态
    [Tags]    中    未评审

    # 1. 查询当前状态
    ${original_status}    商户详情查询    ${merch_no}

    # 2. 变更状态
    商户状态变更    ${merch_no}    暂停

    # 3. 验证变更结果
    ${updated_status}    商户详情查询    ${merch_no}
    Should Be Equal    ${updated_status}[status]    暂停

    # 4. 恢复原始状态
    [Teardown]    商户状态变更    ${merch_no}    ${original_status}[status]
```

**关键点：**
- 直接调用变更接口，无需申请单
- 变更后立即查询验证
- 测试完成后立即恢复原始数据

### 2.2 查询业务

**模式：** 直接查询并验证

```robotframework
*** Test Cases ***
商户详情查询-正常商户
    [Documentation]    【预置条件】已存在正常状态的商户 【操作步骤】查询商户详情，验证返回数据 【预期结果】成功返回商户详情信息
    [Tags]    中    未评审

    # 查询商户详情
    ${detail}    商户详情查询    ${merch_no}

    # 验证返回数据
    Should Not Be Empty    ${detail}
    Should Be Equal    ${detail}[status]    正常
```

### 2.3 终端管理业务

**模式：** 清理-操作-验证-清理

```robotframework
*** Test Cases ***
终端加机-存量终端
    [Documentation]    【预置条件】已存在商户和可用终端 【操作步骤】加机前先撤机清理，执行加机操作，验证结果，撤机恢复 【预期结果】终端加机成功，验证通过后撤销
    [Tags]    中    未评审

    # 1. 加机前清理
    [Setup]    商户加机前查询并撤机    ${device_sn}    ${status_list}

    # 2. 执行加机
    存量终端加机    ${merch_no}    ${device_sn}

    # 3. 验证加机结果
    ${term_info}    商户终端列表查询    ${merch_no}
    Should Contain    ${term_info}[term_no_list]    ${term_no}

    # 4. 清理
    [Teardown]    终端撤机    ${merch_no}    ${term_no}
```

**关键点：**
- 加机前先清理环境，避免重复加机
- 加机后需要等待 3 秒再撤机
- 使用 Teardown 确保环境恢复

## 3. V4.0 商户系统业务模式

### 3.1 变更业务

**模式：** 申请单-提交-审核-验证-恢复

```robotframework
*** Test Cases ***
V4商户状态变更-正常变暂停
    [Documentation]    【预置条件】已存在状态为"正常"的商户 【操作步骤】创建变更申请单，提交状态变更申请，等待审核，验证结果，作废申请单 【预期结果】商户状态成功变更，审核通过后作废申请单
    [Tags]    中    未评审

    # 1. 创建申请单号
    ${apply_id}    V4商户申请单随机生成    ${agent_id}    0

    # 2. 提交状态变更
    V4商户状态变更    ${merch_no}    暂停    ${apply_id}

    # 3. 等待审核（自动审核约10秒）
    Sleep    ${sd_change_auth_sleeptime}

    # 4. 验证变更结果
    ${detail}    V4商户变更申请详情查询    ${apply_id}
    Should Be Equal    ${detail}[status]    审核通过

    # 5. 作废申请单
    [Teardown]    Run Keyword If Test Failed    V4商户申请单状态修改    ${apply_id}    5
```

**关键点：**
- 变更前必须创建申请单号
- 需要传入申请单号和审核信息
- 等待自动审核或使用人工审核
- 用例失败时作废申请单

### 3.2 入网业务

**模式：** 资料提交-审核-验证

```robotframework
*** Test Cases ***
V4营业执照商户入网-正常流程
    [Documentation]    【预置条件】已准备营业执照商户资料 【操作步骤】按模块顺序提交资料，提交审核，等待审核通过 【预期结果】商户入网成功，审核通过
    [Tags]    中    未评审

    # 1. 服务健康检查
    [Setup]    服务健康检查    ${merch_access_standard_node}

    # 2. 生成申请单号
    ${apply_id}    V4商户申请单随机生成    ${agent_id}    1

    # 3. 生成基础信息
    ${base_data}    V4商户基础信息生成    1    ${agent_id}

    # 4. 按模块提交资料
    V4营业执照商户企业类型提交资料    ${base_data}    ${apply_id}

    # 5. 提交审核
    V4商户资料提交审核    ${apply_id}

    # 6. 等待审核
    Sleep    ${sd_sleep_time}

    # 7. 验证审核结果
    ${result}    V4商户变更申请详情查询    ${apply_id}
    Should Be Equal    ${result}[status]    审核通过

    # 8. 作废申请单
    [Teardown]    Run Keyword If Test Failed    V4商户申请单状态修改    ${apply_id}    5
```

**关键点：**
- 需要服务健康检查
- 按模块顺序提交资料（基本信息→身份信息→经营信息→结算信息）
- 入网规则校验在资料提交时自动执行
- 需要等待数据同步（3-5秒）

### 3.3 终端加机业务

**模式：** 申请单-加机-立即撤机

```robotframework
*** Test Cases ***
V4终端加机-码牌终端
    [Documentation]    【预置条件】已存在商户和可用终端 【操作步骤】创建申请单，执行加机，验证结果，立即撤机 【预期结果】终端加机成功，验证通过后撤销
    [Tags]    中    未评审

    # 1. 创建申请单号
    ${apply_id}    V4商户申请单随机生成    ${agent_id}    0

    # 2. 执行加机
    V4终端加机    ${merch_no}    ${apply_id}    ${device_sn}    码牌    ${term_type}    ${term_model}

    # 3. 验证加机结果
    ${term_no}    Resp Dict Value    $.data.term_no
    Should Not Be Empty    ${term_no}

    # 4. 等待数据同步
    Sleep    3s

    # 5. 立即撤机
    [Teardown]    终端撤机    ${merch_no}    ${term_no}
```

**关键点：**
- 加机需要传入申请单号
- 支持多种终端类型（码牌、电子码牌、传统POS、智能POS、码付等）
- 加机后等待 3 秒再撤机
- 撤机确保环境恢复

## 4. 通用技巧总结

### 4.1 数据准备

**存量商户：**
```robotframework
# 使用存量业务关键字准备数据
${merch_no}    存量商户入网    ${agent_id}
```

**V4.0 商户：**
```robotframework
# 使用 V4 商户关键字准备数据
${base_data}    V4商户基础信息生成    ${merch_type}    ${agent_id}
```

### 4.2 异步等待

```robotframework
# 变更类操作等待自动审核（10秒）
Sleep    ${sd_change_auth_sleeptime}

# 入网类操作等待数据同步（根据配置）
Sleep    ${sd_sleep_time}

# 查询类操作等待 2-5 秒
Sleep    3s

# 加机类操作等待数据同步（3秒）
Sleep    3s
```

### 4.3 错误处理

```robotframework
# 申请单清理（用例失败时）
[Teardown]    Run Keyword If Test Failed    V4商户申请单状态修改    ${apply_id}    5

# 数据恢复（存量业务）
[Teardown]    商户状态变更    ${merch_no}    ${original_status}

# 终端清理（加机业务）
[Teardown]    终端撤机    ${merch_no}    ${term_no}
```

### 4.4 响应验证

```robotframework
# 使用 JSONPath 验证特定字段
Resp Dict Compare    {"$.data.status":"暂停","$.ret_code":"00"}

# 提取数组中的特定元素
${item}    Resp Dict Value    $.data[?(@.id=='${target_id}')]

# 多字段验证
Resp Dict Compare    {"$.data.merchant_no":"M001","$.data.status":"active"}
```

## 6. 其他业务模块

商户系统是业务模块之一，其他业务模块（如订单系统、用户系统、支付系统等）应参考本文档结构，创建独立的业务规范文档。

**创建新业务规范文档时，应包含：**
1. 业务模块概述
2. 各子业务模式
3. 通用技巧总结
4. 关键字索引
5. 示例用例
