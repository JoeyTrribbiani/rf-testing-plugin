# TAPD 导入规范

## Excel 列定义

| 列名 | 说明 | 必填 | 示例 |
|--------|------|--------|--------|
| 用例目录 | 用例所属目录，用于组织用例结构 | 否 | [V4.0]商户系统 - 业务接入层 |
| 用例名称 | 用例标题，清晰描述测试内容 | 是 | 商户状态变更测试 - 从正常到暂停 |
| 需求ID | 关联的需求 ID，用于追溯 | 否 | REQ-001 |
| 前置条件 | 执行测试前需要满足的条件 | 否 | 已存在状态为正常的商户 |
| 用例步骤 | 操作步骤描述，分条列出 | 是 | 1. 调用商户状态变更接口<br>2. 提交状态变更为暂停的申请<br>3. 等待审核通过 |
| 预期结果 | 执行后应达到的结果 | 是 | 商户状态成功变更为暂停，可进行暂停状态操作 |
| 用例类型 | 测试类型分类 | 是 | 功能测试 / 性能测试 / 安全测试 / 兼容性测试 |
| 用例状态 | 用例是否有效 | 是 | 正常 / 废弃 |
| 用例等级 | 用例重要性级别 | 是 | 高 / 中 / 低 |
| 创建人 | 用例创建人名称 | 是 | 测试工程师 |
| 是否自动化 | 是否可以自动化执行 | 是 | 是 / 否 |
| 实现自动化 | 是否已实现自动化 | 是 | 是 / 否 |
| 计划自动化 | 是否计划实现自动化 | 是 | 是 / 否 |

## [Documentation] 格式要求

RF 用例的 `[Documentation]` 必须为三段式格式：

```
[Documentation]    【预置条件】已存在状态为正常的商户 【操作步骤】调用商户状态变更接口，提交状态变更为暂停的申请 【预期结果】商户状态成功变更为暂停，可进行暂停状态操作
```

**三段式说明**：

| 段落 | 标签 | 说明 | 示例 |
|------|------|------|------|
| 预置条件 | 【预置条件】 | 执行前的条件 | 已存在状态为正常的商户 |
| 操作步骤 | 【操作步骤】 | 具体操作步骤 | 调用商户状态变更接口，提交状态变更为暂停的申请 |
| 预期结果 | 【预期结果】 | 预期的执行结果 | 商户状态成功变更为暂停，可进行暂停状态操作 |

## [Tags] 标签规范

### 优先级标签

| 标签 | 对应用例等级 | 说明 |
|------|-------------|------|
| `P0` | 高 | 核心功能，必须测试 |
| `P1` | 中 | 重要功能，应该测试 |
| `P2` | 低 | 一般功能，可以测试 |

### 评审状态标签

| 标签 | 对应用例状态 | 说明 |
|------|-------------|------|
| `approved` | 正常 | 已评审通过 |
| `wip` | 正常 | 编写中 |
| `deprecated` | 废弃 | 已废弃 |

### 需求关联标签

| 标签格式 | 说明 |
|---------|------|
| `REQ-*` | 关联的需求 ID，如 `REQ-001` |

### 标签示例

```
[Tags]    P0    approved    REQ-001
```

## 用例类型映射

| RF [Tags] | TAPD 用例类型 |
|-----------|-------------|
| `functional` | 功能测试 |
| `performance` | 性能测试 |
| `security` | 安全测试 |
| `compatibility` | 兼容性测试 |
| 无标签 | 功能测试（默认） |

## Base64 编码

### 为什么要 Base64 编码

TAPD API 导入测试用例时，Excel 文件需要以 Base64 编码字符串形式提交。

### 编码示例

```python
import base64

# 读取 Excel 文件
with open('test_cases.xlsx', 'rb') as f:
    excel_data = f.read()

# Base64 编码
base64_str = base64.b64encode(excel_data).decode('utf-8')

# 保存到文件
with open('test_cases.b64', 'w') as f:
    f.write(base64_str)
```

### API 调用示例

```python
# 假设使用 TAPD MCP
mcp_call("tapd", "import_test_cases", {
    "workspace_id": "48200023",
    "file_base64": base64_str,
    "filename": "test_cases.xlsx"
})
```

## 常见问题

### Q1: Excel 导入后乱码

**原因**: 文件编码问题

**解决**: 确保使用 UTF-8 编码保存 Excel 文件

### Q2: 需求ID 无法关联

**原因**: 需求 ID 格式不正确或需求不存在

**解决**: 检查需求 ID 是否正确，使用 `REQ-XXX` 格式

### Q3: 用例步骤换行显示异常

**原因**: Excel 单元格换行符问题

**解决**: 使用 `<br>` 标记换行，Excel 会自动识别

### Q4: Base64 编码过长

**原因**: Excel 文件过大

**解决**: 考虑分批导入或压缩文件

## 脚本参数说明

### robot2tapd.py

```bash
python robot2tapd.py <robot_file> [options]

参数:
  robot_file            RF 用例文件路径（必填）
  --excel, -e           输出 Excel 路径（默认：同文件名.xlsx）
  --dir, -d             用例目录（默认：当前目录）
  --sheet, -s           Excel 工作表名称（默认：测试用例）
  --creator, -c         创建人（默认：当前用户）
  --out-b64, -b         base64 输出文件（默认：同文件名.b64）
  --batch               批量处理目录（处理指定目录下所有 .robot 文件）
  --help, -h            显示帮助信息
```

### 示例

```bash
# 单个文件转换
python robot2tapd.py test_cases.robot --excel output.xlsx --creator "测试工程师"

# 批量转换
python robot2tapd.py --batch ./cases --output-dir ./output --creator "测试工程师"
```
