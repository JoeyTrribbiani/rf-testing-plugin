### 本步骤内容

整合 `robotframework-rules` 技能，对 RF 用例进行全面检查并自动修复问题。

### 检查与修复清单

#### 1. 目录结构检查（强制）

**标准结构**：
```
<需求名称>_测试套件/
├── Settings.robot
├── Keywords.robot
├── Variables.robot
└── <需求名称>_测试用例.robot
```

**自动修复**：
- 如果文件缺失，自动创建标准模板文件
- 如果所有内容在一个文件，自动拆分为4个文件
- 修正文件间的 Resource 引用关系

#### 2. [Documentation] 标签规范（强制）

**格式要求**：
```
[Documentation]    【预置条件】<描述> 【操作步骤】<描述> 【预期结果】<描述>
```

**必须在一行内完成，禁止换行**

**自动修复**：
- 缺少某个部分时，根据用例内容自动补充
- 格式不正确时，自动调整为标准格式
- 多行时合并为一行

#### 3. 用例命名规范（强制）

**格式**：`业务操作_具体场景`
**分隔符**：必须使用下划线 `_`，**禁止空格**

**自动修复**：
```
"商户状态变更 正常变暂停" → "商户状态变更_正常变暂停"
"申请单信息提交" → "申请单信息提交"  # 无需修改
```

#### 4. 变量命名规范

**规则**：蛇形命名法 `${variable_name}`

**自动修复**：
```
${userName} → ${user_name}
${merchNo} → ${merch_no}
```

#### 5. 关键字命名规范

**存量关键字**：直接使用业务描述
**V4.0关键字**：以 `V4` 或 `V4商户` 开头

#### 6. JSONPath 表达式规范

**基本语法检查**：
- 必须以 `$` 开头
- 属性访问使用 `.`
- 数组索引使用 `[]`
- 过滤器表达式语法正确

#### 7. 标签使用规范

**必填标签**：
- `[Tags]`：优先级（高/中/低）+ 评审状态（未评审/已通过/需修改）
- `[Timeout]`：超时时间（默认 2 minutes）

**自动修复**：
- 缺少 Tags 时，根据用例复杂度自动添加
- 缺少 Timeout 时，默认添加 `2 minutes`

#### 8. 变量作用域规范

| 作用域 | 关键字 | 使用场景 |
|--------|--------|----------|
| 局部变量 | `Set Variable` | 仅在当前关键字或用例步骤内有效，**优先使用** |
| 测试变量 | `Set Test Variable` | 当前测试用例内可见 |
| 套件变量 | `Set Suite Variable` | 当前测试套件内可见 |
| 全局变量 | `Set Global Variable` | 所有测试和套件中可见，**应谨慎使用** |

#### 9. 注释使用规范

- 使用 `Comment` 关键字添加步骤说明
- 描述操作意图而非技术实现
- 在关键步骤前后添加注释说明业务逻辑

### 质量评分

**评分维度**：
- 结构规范性（25分）：目录结构、文件引用
- 命名规范性（25分）：用例名、变量名、关键字名
- 文档完整性（25分）：Documentation、Tags
- 语法正确性（25分）：能 dryrun 通过

**质量门禁**：
- 90-100分：优秀
- 70-89分：良好，通过
- 50-69分：及格，需修复
- <50分：不合格
