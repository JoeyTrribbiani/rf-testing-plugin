# 需求分析技能

> 从需求文档中提取测试点，进行风险评估

## 技能描述

作为测试需求分析专家，能够：
- 解析 PRD 和需求文档
- 提取关键测试点
- 识别风险区域
- 生成测试建议

## 输入格式

```
需求文档（Markdown/PDF/TAPD URL）
```

## 输出格式

```markdown
# 需求分析报告

## 需求概述
[一句话描述]

## 核心功能点
1. [功能点1]
2. [功能点2]

## 测试点提取

### 功能测试
- [ ] 测试点1
- [ ] 测试点2

### 接口测试
- [ ] API测试点

### 边界测试
- [ ] 边界条件

## 风险评估

| 风险项 | 等级 | 说明 |
|--------|------|------|
| [风险1] | 高/中/低 | [说明] |

## 测试建议
1. [建议1]
2. [建议2]
```

## 工具依赖

- `query_knowledge`: 检索历史类似需求
- `query_tapd_stories`: 查询关联需求
- `search_web`: 搜索技术方案

## 示例对话

**用户**: 帮我分析这个需求：https://tapd.xxx.com/story/123

**Agent**: 
1. 使用 `query_tapd_stories` 获取需求详情
2. 使用 `query_knowledge` 检索类似需求
3. 分析并输出测试点和风险评估

_需求分析是测试质量的起点_
