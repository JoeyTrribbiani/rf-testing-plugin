# 测试用例生成技能

> 从测试点生成详细的测试用例

## 技能描述

作为测试用例设计专家，能够：
- 将测试点转换为详细用例
- 覆盖正向、逆向、边界场景
- 生成标准化用例格式
- 支持参数化设计

## 输入格式

```
测试点列表 + 需求文档 + 代码分析结果（可选）
```

## 输出格式

```markdown
# 测试用例

## 用例ID: TC-XXX-001

| 字段 | 内容 |
|------|------|
| 用例名称 | [名称] |
| 所属模块 | [模块] |
| 优先级 | P0/P1/P2/P3 |
| 前置条件 | [条件] |
| 测试步骤 | 1. [步骤1]<br>2. [步骤2] |
| 预期结果 | [预期] |
| 测试数据 | [数据] |

## 用例ID: TC-XXX-002
...
```

## 工具依赖

- `query_knowledge`: 检索历史用例模板
- `get_test_cases`: 查询现有用例
- `create_test_case`: 创建测试用例
- `generate_test_case`: 生成用例代码

## 用例设计原则

1. **MECE原则**: 相互独立，完全穷尽
2. **边界优先**: 重点关注边界条件
3. **可自动化**: 用例步骤清晰，便于自动化

## 示例对话

**用户**: 根据这些测试点生成用例：[测试点列表]

**Agent**:
1. 分析测试点，确定覆盖范围
2. 设计正向、逆向、边界用例
3. 输出标准化用例格式
4. 可选：调用 `create_test_case` 写入TAPD
