# 功能测试能力

## 核心测试类型

### 1. API 测试
**定义**：验证应用程序编程接口是否按预期运行的过程，涉及向 API 发送数据请求并确认响应是否满足可靠性、性能、易用性和安全性预期。

**应用方法与场景**：
- 向 API 端点发送特定请求
- 检查返回的状态码、响应体和数据结构
- 验证 API 在不同输入和场景下的一致性执行
- 测试 API 的功能、性能、安全性和兼容性

**常用工具**：Postman, JMeter, SoapUI, Apifox, Karate, RestAssured, Supertest, Pytest

**能力水平示例**：
- 初级：能使用 Postman 发送基本 HTTP 请求
- 中级：能编写参数化测试用例和断言
- 高级：能构建完整的 API 自动化测试框架并集成到 CI/CD 流水线

### 2. 接口测试
**定义**：不依赖用户界面，直接对系统模块间或系统间的接口进行请求与响应验证，确保数据交互正确性。

**应用方法与场景**：
- 使用 Postman 或 JMeter 发送 HTTP 请求
- 验证接口返回状态码、数据结构及业务逻辑是否符合预期
- 适用于前后端分离架构下的集成验证
- 常用于 CI/CD 流水线中

**常用工具**：Postman, JMeter

**能力水平示例**：
- 初级：能使用 Postman 手动发送接口请求并校验结果
- 中级：能编写参数化脚本实现批量接口测试
- 高级：能构建自动化接口测试框架并与持续集成系统集成

### 3. 数据库测试
**定义**：验证数据库操作的正确性，包括数据的增删改查、事务处理、存储过程等功能。

**应用方法与场景**：
- 编写 SQL 语句验证数据库操作
- 检查数据一致性
- 测试存储过程和触发器

**常用工具**：MySQL, Oracle, SQL Server

**能力水平示例**：
- 初级：能执行基本的 SQL 查询
- 中级：能编写复杂的查询语句
- 高级：能设计数据库测试框架

### 4. 单元测试
**定义**：针对软件中最小可测试单元（如函数、方法）进行验证，确保其逻辑正确性。

**应用方法与场景**：
- 开发人员在编码完成后编写 JUnit 或 pytest 测试用例
- 验证单个模块的功能
- 集成到 CI 流程中实现每次提交自动运行

**常用工具**：JUnit, pytest, NUnit

**能力水平示例**：
- 初级：能为简单函数编写断言测试
- 中级：能使用 mock 技术隔离依赖进行测试
- 高级：能推动 TDD（测试驱动开发）实践落地

### 5. 集成测试
**定义**：将多个已通过单元测试的模块组合起来，测试它们之间的接口与交互是否正常。

**应用方法与场景**：
- 在微服务架构中，测试服务间的 API 调用与数据传递
- 验证数据库连接、消息队列通信等功能集成点
- 适用于系统组件逐步组装阶段

**能力水平示例**：
- 初级：能执行预设的集成测试用例
- 中级：能设计端到端集成测试流程
- 高级：能实施契约测试（如 Pact）保障服务间接口稳定性

### 6. 系统测试
**定义**：对完整的软件系统进行全面测试，验证其功能、性能、安全等是否满足需求规格说明。

**应用方法与场景**：
- 在软件交付前，模拟真实用户环境进行全流程测试
- 包括功能、性能、兼容性、安全等多个维度
- 适用于瀑布或敏捷模型的最终验证阶段

**能力水平示例**：
- 初级：能执行系统测试计划中的用例
- 中级：能独立设计系统级测试方案
- 高级：能协调多团队完成大规模系统联调与压测

### 7. 验收测试
**定义**：由最终用户或客户在真实或模拟环境中进行测试，确认软件是否满足业务需求，决定是否接受该产品。

**应用方法与场景**：
- 在项目交付前，邀请关键用户参与 α 测试
- 发布候选版本供外部用户进行 β 测试
- 适用于产品上线前的最后一道质量关卡

**能力水平示例**：
- 初级：能协助准备验收测试环境与数据
- 中级：能组织并引导用户完成验收测试
- 高级：能制定验收标准并推动签署交付协议

### 8. 回归测试
**定义**：在软件修改或更新后重新执行已有测试用例，确保原有功能未被破坏。

**应用方法与场景**：
- 每次代码提交后运行自动化回归测试套件
- 版本迭代后对核心功能进行手工回归验证
- 适用于所有变更后的质量保障活动

**能力水平示例**：
- 初级：能执行既定的回归测试用例
- 中级：能筛选受影响模块进行针对性回归
- 高级：能构建智能回归测试体系，按变更影响动态调整测试范围
