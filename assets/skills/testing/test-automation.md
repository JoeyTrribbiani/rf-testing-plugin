# 自动化测试能力

## 核心能力

### 1. 自动化测试
**定义**：通过编写脚本或使用工具自动执行测试任务，替代重复性手工操作，提高测试效率和回归覆盖能力。

**应用方法与场景**：
- 在敏捷开发中，利用 Selenium 或 Appium 实现 Web 或 APP 的 UI 自动化测试
- 结合 pytest 或 TestNG 框架实现接口自动化
- 适用于高频回归测试场景

**能力水平示例**：
- 初级：能录制并回放简单 UI 操作脚本
- 中级：能编写可维护的自动化测试脚本并集成 CI
- 高级：能设计分层自动化测试框架，支持数据驱动与关键字驱动

### 2. API 自动化测试
**定义**：熟练使用 Postman、RestAssured 等工具，对 RESTful API、GraphQL、gRPC 等协议进行自动化测试，确保接口功能、性能与安全性的可靠实现。

**应用方法与场景**：
- 构建完整的自动化测试流水线并集成到 DevOps 流程

**常用工具**：Postman, RestAssured, pytest, Jenkins, GitLab CI

**能力水平示例**：
- 初级：能使用 Postman 进行简单 API 测试
- 中级：能编写 Python/Java 测试脚本
- 高级：能构建完整的自动化测试流水线并集成到 DevOps 流程

### 3. UI 自动化测试
**定义**：通过编写脚本模拟用户操作，以可重复的方式高效验证应用行为，检查回归问题及设备兼容性，提高测试效率和覆盖率。

**应用方法与场景**：
- 在持续集成环境中运行自动化测试套件
- 执行视觉回归测试，比较 UI 变化
- 针对回归测试高频场景进行自动化验证
- 适用于迭代频繁的敏捷开发项目

**常用工具**：Selenium, Cypress, Appium, Percy, Applitools Eyes

**能力水平示例**：
- 初级：能录制简单操作脚本
- 中级：能编写参数化测试用例
- 高级：能构建可维护的自动化测试框架并进行跨浏览器兼容性测试

### 4. 测试框架深度掌握
**定义**：精通各种前端、移动端、API 和性能测试框架，具备搭建、优化和扩展测试框架的能力，以适应复杂多变的项目需求。

**应用方法与场景**：
- 在项目中选择合适的测试框架，进行框架搭建和优化
- 提高测试效率和质量

**能力水平示例**：
- 初级：能使用框架
- 中级：能配置框架
- 高级：能优化和扩展框架

### 5. 持续集成与交付
**定义**：掌握 CI/CD 流水线集成能力，将自动化测试嵌入到代码提交、构建、部署的全过程，实现快速反馈和质量门禁控制。

**应用方法与场景**：
- 配置自动化测试任务的触发条件
- 实现代码提交后自动执行测试，确保代码质量
- 结合容器化技术（Docker/K8s）实现环境的快速部署与治理

**常用工具**：GitLab CI, Jenkins, GitHub Actions, CircleCI

**能力水平示例**：
- 初级：能配置基础流水线
- 中级：能管理 CI/CD
- 高级：能优化交付效率

### 6. 质量门禁
**定义**：在 CI/CD 管道中设立的测试质量阈值，只有通过特定测试覆盖率和通过率的构建才能进入下一阶段，确保发布质量。

**应用方法与场景**：
- 在 CI/CD 管道中设置智能质量门禁
- 建立多维质量指标如性能基线门禁、安全扫描门禁、用户体验门禁
- 根据部署环境差异化设置阈值

**能力水平示例**：
- 初级：能配置基础的质量门禁规则
- 中级：能设计多维度质量门禁体系
- 高级：能实现智能质量门禁并根据环境动态调整阈值

### 7. 测试左移实践
**定义**：将测试活动前移到软件开发生命周期的早期阶段，包括需求评审、设计评审和开发阶段，通过早期发现缺陷来降低修复成本。

**应用方法与场景**：
- 在需求评审阶段参与需求分析和测试用例设计
- 在开发阶段实施单元测试和代码审查
- 在持续集成过程中执行自动化测试

**能力水平示例**：
- 初级：能参与需求评审会议，提出测试角度的问题
- 中级：能编写基础的单元测试用例，参与 CI 流水线配置
- 高级：能设计完整的测试左移策略，推动质量文化建立

### 8. 测试策略设计
**定义**：制定分层测试策略，合理规划单元测试、集成测试、端到端测试的比例与优先级，最大化测试投资回报率。

**应用方法与场景**：
- 根据项目特点设计测试策略
- 确定测试类型、测试范围和测试优先级
- 采用"测试金字塔"模型，强调底层自动化测试的高覆盖率

**能力水平示例**：
- 初级：能制定基本策略
- 中级：能优化策略
- 高级：能设计完整测试体系

## Robot Framework 自动化测试

对于 Robot Framework（RF）自动化测试的具体实现，请加载以下技能：

| 技能 | 用途 | 触发关键词 |
|------|------|-----------|
| `rf_basics` | RF 基础规范与用例结构 | robot framework, rf, 用例, Setup, Teardown |
| `rf_keywords` | RF 自定义关键字编写 | 关键字, 自定义关键字, documentation, 定义 |
| `rf_jsonpath` | RF JSONPath 表达式与数据提取 | jsonpath, 响应, 校验, 提取, data, 字段 |
| `rf_variables` | RF 变量管理与作用域 | 变量, 套件变量, @{}, ${}, 作用域 |
| `rf_validation` | RF 响应验证与断言 | 验证, 断言, should, 校验, 响应校验 |
| `rf_builtin` | RF 内置关键字用法 | builtin, 内置关键字, log, run keyword |
