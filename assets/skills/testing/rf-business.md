# Robot Framework · 商户系统业务模式

## 变更业务

### 存量流程（三段式）
1. **修改** - 调用变更接口修改商户信息
2. **验证** - 查询验证修改结果
3. **恢复** - 将数据恢复原值

### V4 流程（五段式）
1. **申请单** - 使用 `V4商户创建变更申请单` 或随机生成申请单号
2. **提交** - 提交变更申请
3. **审核** - 使用 `V4商户通用审核` 进行审核
4. **验证** - 查询验证变更结果
5. **恢复** - 将数据恢复原值

**技巧**：
- 变更后通常等待 10 秒（自动审核）
- 失败时使用 `V4商户申请单状态修改` 作废申请单
- 使用 `Run Keyword If Test Failed` 清理申请单

## 查询业务

- 直接查询并校验结果
- 使用 `Resp Dict Compare` + JSONPath 做多字段校验
- 使用 `Should Not Be Empty`、`Should Not Be Equal` 等断言
- 查询类关键字通常设置套件变量

## 加机业务

### 存量
- 前置：`商户加机前查询并撤机`
- 执行：`存量终端加机`
- Teardown：`存量终端撤机`

### V4
- Setup：使用 `Run Keyword And Ignore Error` 清理
- 加机后立即撤机
- 加机后等待 3 秒再撤机

## 报备业务

- 调用报备接口 → 查库验证记录
- 微信/支付宝需先实名认证
- 报备后等待 3-5 秒再查库
- 使用 `通用查询数据库` 验证写入

## 入网业务

### 存量
- 电签：`电签首次入网`
- 营业执照：`合伙人营业执照商户首次入网`
- 小微：`合伙人小微商户入网`

### V4
- `V4商户申请单随机生成`
- `V4商户基础信息生成`
- 按模块提交（基本信息 → 身份 → 经营 → 结算 → 其他）
- 入网后等待 3-5 秒

## 进件资料

- 申请单 + 基础信息生成
- 按模块提交
- 规则校验
- `服务健康检查`、加密字段处理

## 通用原则

- 优先用现有接口查数据，避免直接查库
- 修改存量用例前先理解原逻辑
- 不随意改 Variables.robot 已有变量
