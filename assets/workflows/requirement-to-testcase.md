# Workflow: 需求到测试用例

> 从需求分析到测试用例生成的完整工作流

## 流程图

```
需求输入 → 需求解析 → 测试点提取 → 用例设计 → 用例输出
    ↓           ↓           ↓           ↓
  TAPD URL   结构化清单   测试点清单   测试用例
```

## 步骤定义

### Step 1: 需求输入
**输入**: 
- TAPD需求URL
- 或PRD文档（Markdown/PDF）

**处理**:
- 调用 TAPD 工具获取需求详情
- 或解析上传的文档

**输出**: 需求原始数据

---

### Step 2: 需求解析
**技能**: requirements-analysis

**处理**:
- 提取功能模块
- 识别业务规则
- 发现边界条件

**输出**: 结构化需求清单

---

### Step 3: 测试点提取
**技能**: requirements-analysis

**处理**:
- 功能测试点（正向/逆向）
- 边界测试点
- 接口测试点
- 性能测试点

**输出**: 测试点清单

---

### Step 4: 用例设计
**技能**: test-case-generation

**处理**:
- 等价类划分
- 边界值分析
- 场景设计

**输出**: 测试用例

---

### Step 5: 用例输出
**处理**:
- 格式化为标准模板
- 可选：导入TAPD
- 可选：生成Robot Framework脚本

**输出**: 
- Markdown测试用例文档
- 或TAPD测试用例
- 或RF自动化脚本

---

## 使用方式

```bash
# 触发工作流
"帮我分析这个需求并生成测试用例：https://www.tapd.cn/xxx"

# 或
"分析这份PRD并生成测试用例" [上传文档]
```
